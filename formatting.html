Formatting Rules

Formatting must be consistent across all projects.

HTML

  • Indent: 2 spaces
  • Always close tags
  • Boolean attributes without value: disabled
  • Use double quotes for attributes

CSS

  • Use rem for typography
  • Sort properties logically (layout → typography → visual)
  • Always include a newline between selectors

JavaScript

  • Use ES modules if applicable
  • No var, only let / const
  • Arrow functions preferred

VS Code: Live Sass Compiler Setup

Our team uses the VS Code extension Live Sass Compiler (Glenn Marks) to compile SCSS into CSS. All compiled CSS must always be output to the project’s css/ directory. SCSS files are used only locally and must not be committed to the public GitHub guideline repository.

SCSS Workflow Rules

  • Write styles in .scss files locally.
  • Use Live Sass Compiler (Glenn Marks) to compile them.
  • Compiled CSS must be saved into /css or /assets/css depending on the project structure.
  • Never edit compiled CSS directly. Always edit SCSS.
  • GitHub (guideline repo): only compiled CSS is stored. SCSS files are not uploaded.

Note (Japanese)

  • SCSS を直接 GitHub（ガイドライン）にアップロードしない。
  • 変更は SCSS に加え、コンパイルしてから CSS をコミット。
  • CSS を⼿動で編集しない（常に SCSS → CSS）。

Suggested Live Sass Compiler Settings

These are conceptual settings showing how to configure output folder: 

{
  // Output CSS into /assets/css
  "liveSassCompile.settings.formats": [
    {
      "format": "expanded",
      "extensionName": ".css",
      "savePath": "/assets/css"
    }
  ],

  // Auto-compile on save (optional)
  "liveSassCompile.settings.autoprefix": [],
  "liveSassCompile.settings.generateMap": false
}

You can adjust savePath depending on project structure, but the rule stays the same: Compiled CSS always belongs in the CSS directory.

Checklist

  • Live Sass Compiler installed (Glenn Marks version).
  • Output directory = css/.
  • No SCSS committed to GitHub guideline repository.
  • Do not modify compiled CSS manually.
  • Commit only compiled CSS after changes.
